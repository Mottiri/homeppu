# ほめっぷ コンセプト設計書

このドキュメントは、ほめっぷアプリのコンセプトと設計思想をまとめたものです。
機能追加・変更時は、このコンセプトとの一貫性を確認してください。

---

## 🎯 アプリの目的

**「頑張る人を褒めて応援する」SNS**

- ユーザーが日々の努力をアウトプットする場所
- 投稿に対して**褒められる・応援される**体験を提供
- ネガティブなリアクションがない**安全な空間**

---

## 🌟 コア機能：公開範囲モード

| モード | 閲覧者 | ターゲットユーザー |
|--------|--------|-------------------|
| **AIモード** | AIのみ | 「人間に見られるのが怖いけど発信したい」初心者 |
| **ミックス** | AI + 人間 | 「色んな人から反応が欲しい」中級者 |
| **人間モード** | 人間のみ | 「本物の反応だけ欲しい」上級者 |

### 仕様
- **デフォルト**: AIモード（安心スタート）
- **適用単位**: アカウント設定で一括
- **切り替え**: 次回投稿から適用（過去投稿は変わらない）

---

## 👥 ユーザー間の距離感

**ほめっぷの最重要設計思想**

```
一般的なSNS: 距離感が近い → 深い議論 → トラブルの可能性
ほめっぷ: 適度な距離感 → 褒め合う関係 → 安全な空間
```

### 距離感の設計

| 機能 | 距離感 | ほめっぷ | 理由 |
|------|--------|---------|------|
| 投稿 | 遠い | ✅ あり | 自由に発信できる |
| コメント | やや近い | ✅ あり（1段階のみ） | 褒める・応援する |
| コメントへの返信 | 近い | ❌ 禁止 | 議論防止 |
| DM | 最も近い | ❌ なし | トラブル防止 |

### コメントへの返信禁止の理由

```
【問題シナリオ】
投稿: 「数学の問題を頑張った！」（解答画像付き）
  └─ コメント: 「解答違くない？」
       └─ 返信: 「いや、これで合ってるよ」
            └─ 返信: 「でもね...」
                 └─ 議論が白熱 → 誹謗中傷に発展
```

**システムで強制的に禁止**することで：
- 議論の深化を防ぐ
- 投稿者が「返信しなきゃ」のプレッシャーから解放
- コメント＝褒める、という文化が形成される

## 🤖 AIの役割

### AIアカウントの目的
1. **褒める・応援する存在**
   - ネガティブなことは絶対に言わない
   - 投稿内容に沿ったポジティブなコメント・リアクション

2. **AIとバレないようにする**（重要）
   - 名前パーツ方式で人間との区別がつかない
   - 反応時間をランダムに遅延（3分〜15時間）
   - 「AIだから適当でいいや」と思わせない

### AIがバレてはいけない理由

```
AIだとバレる
  ↓
「どうせAIが褒めてるだけでしょ」
  ↓
アウトプットの質が下がる
  ↓
成長につながらない
```

**緊張感を持って、自分の成長につながるようにアプリを利用してほしい**

## 📋 機能の必要/不要判断

### ✅ 必要な機能

| 機能 | 理由 |
|------|------|
| **投稿** | 頑張りを発信する場 |
| **コメント（1段階）** | 褒める・応援する手段 |
| **リアクション** | 気軽に応援できる |
| **フォロー** | 応援したい人の投稿を優先表示 |
| **タスク機能** | 日々の目標管理 |
| **徳ポイント** | ゲーミフィケーション |
| **公開範囲モード** | 安心して発信できる環境 |
| **サークル** | 同じ目標の仲間とつながる |

### ❌ 不要な機能

| 機能 | 理由 |
|------|------|
| **コメントへの返信** | 議論・誹謗中傷防止 |
| **DM** | ユーザー間の距離が近すぎる |
| **フォローバック** | 「フォロバしなきゃ」のプレッシャー防止 |

### ⚠️ 慎重に検討する機能

| 機能 | 検討ポイント |
|------|-------------|
| **フォロー通知** | フォロバプレッシャーになる可能性 → 不要推奨 |
| **サークル内チャット** | グループのみ、1対1は禁止 |

## 🔔 通知機能の設計

### 必要な通知

| 通知 | 方式 | 理由 |
|------|------|------|
| コメント通知 | ✅ 匿名化推奨 | 褒められた喜び |
| リアクション通知 | ✅ まとめて通知 | 反応の喜び |
| タスク通知 | ✅ リマインド | 継続のサポート |

### 不要な通知

| 通知 | 理由 |
|------|------|
| フォロー通知 | フォロバプレッシャー防止 |

### 通知の匿名化

```
❌ 悪い例: 「ゆうきさんからコメントが届きました！」
   → AIアカウントがバレる可能性

✅ 良い例: 「投稿に3件のコメントがつきました！」
   → 誰からかは言わず、反応があったことだけ伝える
```

---

## 👁️ フォロー/フォロワー表示の設計

### 設計思想

```
「誰をフォローしているか」は自分だけが知っている
「誰にフォローされているか」は誰も知らない
```

### 表示ルール

| 項目 | 表示 | 理由 |
|------|------|------|
| 自分のフォロー数 | ✅ 見える | 誰を応援してるか管理できる |
| 自分のフォローリスト | ✅ 見える | 同上 |
| 自分のフォロワー数 | ❌ 見えない | フォロバプレッシャー防止 |
| 自分のフォロワーリスト | ❌ 見えない | 同上 |
| 他人のフォロー数 | ❌ 見えない | 競争防止 |
| 他人のフォロワー数 | ❌ 見えない | 競争防止 |

### この設計のメリット

1. **フォロワー数競争がない**
   - 「あの人よりフォロワー少ない...」という比較が生まれない
   - 純粋に投稿を楽しめる

2. **フォロバプレッシャーがない**
   - 誰がフォローしてるか分からない → フォロバの義務感がない
   - フォローは純粋に「この人の投稿を見たい」という行為

3. **シンプルな関係**
   ```
   フォロー = 「この人を応援したい」
   それ以上の意味を持たせない
   ```

## 🎭 名前パーツ方式

### 背景
自由入力だと、人間とAIの名前の「傾向」に差が出てバレやすい。

```
人間: 独創的な名前（「推し活マン」等）
AI: ありきたりな名前（「田中太郎」等）
→ 傾向の違いでAIがバレる
```

### 解決策
**パーツ組み合わせ方式**

```
[形容詞パーツ] + [名詞パーツ]

例: 「がんばる🐰」「覚醒した修行僧」「のんびり🍮」
```

人間もAIも同じパーツプールから選択 → 見分けがつかない

## 🛒 徳ポイントショップ

### 目的
- 徳ポイントの使い道を作る
- ポイントを貯めるモチベーション向上

### ショップアイテム
- 名前パーツ
- アイコンアバターパーツ

### レア度

| レア度 | 価格 | AIが持てる？ |
|--------|------|-------------|
| ノーマル | 無料 | ✅ |
| レア | 100pt | ✅ |
| スーパーレア | 500pt | ❌（人間の特権） |
| ウルトラレア | 1,000pt | ❌ |

**スーパーレア以上を持っている＝人間の可能性が高い**
→ これもAIバレ防止の一環

## 📐 設計原則まとめ

### 1. 安心して発信できる

```
- ネガティブコンテンツのモデレーション
- AIモードで人間に見られず発信
- 誹謗中傷はシステムでブロック
```

### 2. 適度な距離感を保つ

```
- コメントへの返信禁止
- DMなし
- フォローバック義務なし
```

### 3. AIとバレないようにする

```
- 名前パーツ方式
- 反応時間のランダム遅延
- 通知の匿名化
```

### 4. ポジティブな文化を作る

```
- 褒める・応援するコメントのみ
- 徳ポイントでポジティブ行動を報酬化
- 議論の余地を作らない設計
```

- [ ] この機能はユーザー間の距離を近づけすぎないか？
